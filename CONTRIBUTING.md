# Contributing to StreamKernel

Thank you for your interest in StreamKernel.

## Philosophy
StreamKernel values clarity, correctness, performance, and security.

## Feedback

Issues, bug reports, documentation feedback, and reproducibility notes are
welcome.

## Code Contributions

StreamKernel is currently not accepting external code contributions except by
prior written agreement. This keeps licensing, patent, provenance, and
commercial-use boundaries clear while the project is source-available.

If a contribution is accepted under a written agreement, please:

- Prefer small, focused changes
- Respect architectural boundaries
- Keep hot paths allocation-light
- Maintain fail-closed security defaults

## Licensing

By submitting any accepted contribution, you agree that it may be incorporated
into StreamKernel under the license terms that apply to the target file or
module, including the StreamKernel Source Available License (SSAL) where
applicable, unless a separate written agreement says otherwise.

Do not submit patent-sensitive material, confidential customer information, or
third-party code unless you have the right to contribute it under the target
module's license.

## Questions
Open a discussion if unsure where something fits.
